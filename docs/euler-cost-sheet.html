What is a "turn"?

A turn is one complete back-and-forth: the student asks something, Euler explains. That's one turn. A typical session has 25 turns.

What is "prompt caching"?

Every turn sends Euler's big system prompt + prior conversation to Claude. Most of it doesn't change between turns.

Anthropic lets us mark this "static" part as cached. On repeat sends, we pay only 10% of the input rate for cached tokens — a 90% discount on the repeated material.

  • In production, about 65% of input tokens hit the cache (measured).
  • Cache lives for 5 minutes, which fits our use (turns come every ~2 min).
  • Caching is on right now in production.

The caching savings

Prompt caching cuts our cost by ~49% across all models. On Opus 4.6, that's $4.59 saved per session.

At 1,000 sessions/month that's ~$4,600/month saved just from caching being turned on.

Why we use Opus (not cheaper models)

  • Teaching quality. Opus produces clearer explanations with stronger conceptual framing. Sonnet occasionally skips steps.
  • Animations. The 3D/p5 animations Euler draws on the board only render reliably when Opus writes them. Sonnet has a higher failure rate on rendered code.
  • Multi-step reasoning. Planning a lesson across 20+ turns, knowing what to teach next, and adapting to the student is where Opus shines.

Sonnet is a valid downgrade for simpler subjects if cost becomes a concern — cuts cost by 40% while keeping decent quality.

Bottom line

A typical 25-turn voice session costs $6.51 (Opus 4.6 + ElevenLabs Turbo).

Of that: $4.71 is LLM (Anthropic via OpenRouter) + $1.80 is voice (ElevenLabs).

Prompt caching already saves us ~49% on the LLM side. Moving to Sonnet would save another 40%; Haiku saves 80%.
