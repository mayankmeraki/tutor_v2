Capacity: Live AI Teaching

Not a chatbot. A tutor that draws, speaks, adapts, and assesses — in real time.

1

Live Teaching Board

The tutor draws diagrams, writes equations (KaTeX), annotates code, shows images, and builds visual explanations step-by-step on a real-time canvas — just like a whiteboard session with a human tutor.

2

Voice-First Interaction

The tutor speaks with natural TTS, paces explanations with synchronized board drawing, and listens via microphone. Students can interrupt, ask questions, or request re-explanation mid-lesson.

3

Adaptive Lesson Plans

An AI planning agent structures each session with objectives, topics, and checkpoints. The plan adapts in real-time based on student performance and questions.

4

Inline Assessments

MCQ, free-text, fill-in-the-blank, spot-the-error, confidence checks, teach-back, and canvas derivation exercises — woven naturally into the teaching flow, not bolted on.

5

Video Follow-Along

Pair any YouTube lecture or uploaded video with an AI tutor. The tutor knows the transcript and syllabus context, pauses when you have questions, and resumes seamlessly.

6

Interactive Simulations

Course-linked simulations with predict-observe-explain pedagogy. Students make predictions, run the sim, then the tutor explains discrepancies.

03 — How It Works

Multi-Agent Architecture

A coordinated system of specialized AI agents that handle different aspects of the teaching experience.

Tutor Agent

Core teaching engine. Generates board content, voice beats, inline assessments, and manages the active teaching flow. Equipped with 20+ tools.

Planning Agent

Auto-spawns on session start. Structures lessons into topics, sets objectives, manages checkpoints, and adapts the plan based on student progress.

Assessment Agent

Handles dedicated assessment checkpoints. Generates questions matched to taught concepts, evaluates responses, and reports mastery levels.

Content Agent

Searches and retrieves curriculum content, enriched sections, concepts, and student-uploaded materials (BYO) to ground teaching in source material.

Research Agent

Web search, image search, and external knowledge retrieval for topics beyond the indexed curriculum.

Delegation Agent

Sub-agent for bounded drills. The tutor delegates focused practice problems, then the student returns to the main session flow.

05 — Content Infrastructure

Structured Knowledge Base

Courses are deeply structured with semantic search, vector embeddings, and section-level granularity.

Content Hierarchy: Course → Module → Lesson → Timed Section → Concepts

Each lesson has video URLs, transcripts, duration, and enriched section content. Each section maps to a timestamp range in the video for precise follow-along grounding.

Semantic Search

MongoDB Atlas vector search with OpenAI embeddings + Haiku re-ranking. Students find relevant content by asking natural language questions, not browsing menus.

Intent Resolution

Free-text queries like "teach me derivatives" are resolved to specific courses, lessons, and starting sections via the /resolve-course API.

Video Extraction

YouTube URLs are converted to direct stream URLs via yt-dlp, enabling custom playback controls, speed adjustment, and transcript alignment.

Concept Mapping

Per-course concept hierarchies with prerequisites and relationships. Enables mastery-based progression and targeted review.

Student Intelligence

Persistent student model tracks mastery across sessions and concepts.

A

Concept Mastery Vectors

Per-student, per-concept notes and mastery tags stored in MongoDB with vector search. The tutor queries student knowledge before teaching to avoid redundancy and target gaps.

B

Session Continuity

Every session preserves full transcript, plan state, board frames, assessment scores, and backend tutor state. Resume any session exactly where you left off.

C

AI-Generated Artifacts

Flashcards with spaced repetition stats, study plans, and revision notes — automatically generated from teaching sessions and accessible for later review.
