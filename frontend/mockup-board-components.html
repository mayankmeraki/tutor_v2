The product of transcription
DNA gets read and a messenger copy — mRNA — is built. The "m" stands for messenger: it carries a message from DNA to the protein-building machinery.
m = messenger — it carries a MESSAGE from DNA
Binary search invariant
At every iteration, the target — if it exists — is somewhere in nums[lo..hi]. The loop shrinks this window by half each step. When the window is empty (lo > hi), the target isn't in the array.
loop: lo ≤ hi · narrow: mid±1 · terminate: lo > hi → not found
Time complexity of binary search
Each iteration cuts the search space in half. Starting from n elements: n → n/2 → n/4 → ... → 1. That's log₂(n) steps. For 1 billion elements, only ~30 comparisons.
O(log n) — halving each step. 1 billion elements = ~30 comparisons.
Backpropagation intuition
Forward pass: input flows through layers to produce a prediction. Loss measures how wrong the prediction is. Backprop: the loss signal flows BACKWARDS through the network, telling each weight "this is how much you contributed to the error — adjust by this much."
forward = predict · loss = how wrong · backward = blame each weight

Binary Search

The bulletproof template

The idea
Given a sorted array, find a target by repeatedly cutting the search space in half. Each step eliminates 50% of the remaining elements.
O(log n) — 1 billion elements → ~30 comparisons
Compare mid
→match?
Return mid
mid too small
→
lo = mid + 1
mid too big
→
hi = mid - 1
3 invariants
burn these in
lo ≤ hi — loop while at least one element remains
mid = lo + (hi - lo) // 2 — overflow-safe midpoint
lo = mid + 1 / hi = mid - 1 — NEVER just mid (infinite loop)
⚠️The #1 bug
Using while lo < hi instead of while lo <= hi. On a single-element array [42], lo == hi == 0 — the loop never runs and the element is never checked.
