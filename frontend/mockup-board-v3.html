The #1 Binary Search Bug
Bug
while lo < hi:
  mid = (lo + hi) // 2
  if nums[mid] == target:
    return mid
  elif nums[mid] < target:
    lo = mid + 1
  else:
    hi = mid - 1
Fixed
while lo <= hi:
  mid = lo + (hi - lo) // 2
  if nums[mid] == target:
    return mid
  elif nums[mid] < target:
    lo = mid + 1
  else:
    hi = mid - 1
< skips the last element. <= checks it. On [42], the loop must run when lo == hi == 0.

Transcription — The Copying Step
DNA holds the instructions. But it never leaves the nucleus. So the cell makes a disposable copy — mRNA — and sends that out instead.
DNA
stays in the nucleus
read & copy
mRNA
the photocopy — disposable
exits nucleus
Ribosome
reads mRNA, builds protein
Why not send DNA directly?
DNA is the only copy of the instructions. If it gets damaged outside the nucleus, the cell is done. mRNA is cheap — make it, use it, destroy it.
What makes mRNA different?
Single-stranded (DNA is double). Uses U instead of T. Much shorter — just one gene, not the whole genome.
m = messenger — it carries a MESSAGE from DNA
Binary Search — The Template
def binary_search(nums, target):
  lo, hi = 0, len(nums)-1
  while lo <= hi:
    mid = lo + (hi-lo)//2
    if nums[mid] == target:
      return mid
    elif nums[mid] < target:
      lo = mid + 1
    else:
      hi = mid - 1
  return -1
Inclusive bounds — lo and hi both point to valid elements
<= not < — so we check the last remaining element
Overflow-safe midpoint — lo + (hi-lo)//2 never overflows
Always mid±1 — never just mid, or the loop spins forever
