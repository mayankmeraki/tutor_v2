Design Decisions & Mechanics

Checkpoint Map (left tab)

  • Auto-generated from <teaching-heading> tags already in the stream
  • Sections = section headings, items = topic headings
  • Click → scrolls chat to that block (smooth scroll + brief highlight)
  • Active checkpoint tracks scroll position (intersection observer)
  • Done/active/upcoming states mirror the teaching plan progress
  • Shows time offset from session start for each checkpoint

Quick Notes (right tab)

  • New <teaching-note> tag — tutor emits as it teaches
  • Attributes: title, category (gate/concept/formula/rule), content
  • Cards use board font (Caveat) for formulas — feels like scribbled notes
  • Grouped by topic section, most recent at top within group
  • Flash animation when new note arrives (draws attention)
  • Click a note card → scrolls to where it was generated in chat
  • Persisted to session state → restored on page reload

What Tutor Emits

  • <teaching-note title="H Gate" category="gate">
  • Content: plain text + LaTeX formulas between $ signs
  • Tutor generates a note after explaining each key concept
  • Prompted to include: the formula, one-line rule, key properties
  • NOT a teaching action — invisible in chat, only appears in rail
  • One note per concept — new note replaces old if same title

Collapse & Responsiveness

  • Rail width: 260px expanded, 40px collapsed (icon-only)
  • Starts expanded on desktop, collapsed on mobile (<768px)
  • Toggle button persists preference in localStorage
  • When collapsed, icon-only buttons for Map 🗺 and Notes 📝
  • No content lost — just hidden, state preserved
  • Right sidebar unchanged — still has plan + session stats
Teaching Note Tag Spec
Tag format:
<teaching-note title="H Gate" category="gate">
$H = \frac{1}{\sqrt{2}}\begin{pmatrix}1 & 1 \\ 1 & -1\end{pmatrix}$
Creates superposition from classical states
H|0⟩ = |+⟩, H|1⟩ = |−⟩
Self-inverse: H² = I
</teaching-note>

Categories: gate, concept, formula, rule, definition, visual
Rendering: Does NOT appear in chat stream. Appears only in the Notes tab of the study rail.
Upsert: If a note with the same title already exists, it gets replaced (latest wins).
Interaction Flow
1. Tutor explains X gate → emits <teaching-note title="X Gate" category="gate">...</teaching-note>
2. Note card appears in rail Notes tab with a brief flash animation
3. Checkpoint "X Gate — quantum NOT" appears in Map tab (from heading already emitted)
4. Student progresses to H gate → new note + checkpoint added
5. Tutor asks "What does X|+⟩ give?" → student glances at X Gate note on the left, sees the swap rule
6. Student clicks "X Gate" checkpoint in Map tab → chat scrolls back to that section
7. Student clicks back to "H Gate" checkpoint → chat scrolls forward to current position
