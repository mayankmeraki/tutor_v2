Binary Search — The Template

Here's the bulletproof template you should commit to memory. Look at it for a second, then we'll trace through it together.

PYTHON binary_search.py
1def binary_search(nums, target): 2 lo, hi = 0, len(nums) - 1 # inclusive bounds 3 while lo <= hi: 4 mid = lo + (hi - lo) // 2 # overflow-safe midpoint 5 if nums[mid] == target: 6 return mid 7 elif nums[mid] < target: 8 lo = mid + 1 # skip mid, search right 9 else: 10 hi = mid - 1 # skip mid, search left 11 return -1

Three things to burn into memory: lo <= hi (check the last element), mid = lo + (hi - lo) // 2 (overflow-safe), and lo = mid+1 / hi = mid-1 (never just mid — that's an infinite loop waiting to happen).

Most binary-search bugs come from the wrong loop condition. The classic mistake looks like this:

PYTHON
1while lo < hi: # misses the last element 2 mid = (lo + hi) // 2 3 ...

It looks fine — and for arrays of length two or more it usually works. But on a single-element array, lo == hi == 0 from the start, the loop never runs, and the function returns -1 for an element that's right there. Always use <=:

PYTHON
1while lo <= hi: # inclusive — checks the last element 2 mid = lo + (hi - lo) // 2 # overflow-safe midpoint 3 ...
